GAMBARAN KESELURUHAN

PT Chatbot ialah sistem chatbot AI yang direka khusus untuk Pendakwah Teknologi Solutions — sebuah syarikat transformasi digital dan latihan. Sistem ini menggunakan pendekatan RAG (Retrieval-Augmented Generation) dengan pencarian hibrid, pengembangan pertanyaan automatik, dan penilaian kendiri untuk memastikan jawapan berkualiti tinggi.

ALIRAN PEMPROSESAN (PIPELINE)

Setiap soalan pengguna melalui 8 langkah pemprosesan sebelum jawapan dihasilkan:

Klasifikasi Pertanyaan
Sistem mengkategorikan soalan sebagai internal (dokumen PT), external (maklumat web), atau hybrid (kedua-duanya). Menggunakan kata kunci dan corak regex untuk pengesanan tepat.
classify_query() → "internal" | "external" | "hybrid"
Pengembangan Pertanyaan (Query Expansion)
Model LLM pantas (GPT-4o-mini) menjana 3 variasi soalan asal untuk meningkatkan penemuan semula.
GPT-4o-mini → 3 variasi + soalan asal = 4 pertanyaan
Pencarian Hibrid (Vector + BM25)
Dua kaedah pencarian dijalankan serentak: Vector search menggunakan embeddings untuk makna semantik, dan BM25 menggunakan padanan kata kunci. Kedua-dua set hasil digabungkan menggunakan Reciprocal Rank Fusion (RRF).
ChromaDB HNSW (M=32, ef=200) + BM25Okapi → RRF fusion
Carian Web (jika berkaitan)
Untuk pertanyaan external atau hybrid, sistem mencari web menggunakan dwi-pembekal: Tavily (utama) dan Brave Search (sandaran). Pertanyaan difokuskan kepada topik PT secara automatik.
Tavily API (advanced) → Brave fallback → top 5 hasil
Penarafan Semula (Cross-Encoder Reranking)
Cross-encoder menilai setiap pasangan (soalan, dokumen) secara langsung untuk menentukan relevansi sebenar. Menapis daripada 15 calon kepada 7 dokumen terbaik.
ms-marco-MiniLM-L-6-v2 on GPU → top 7 reranked
Penjanaan Jawapan (LLM Generation)
Model utama GPT-4o menjana jawapan berdasarkan dokumen yang diperoleh, dengan arahan Chain-of-Thought (CoT) untuk penaakulan berstruktur. Jawapan distrim secara langsung melalui SSE.
OpenAI API (GPT-4o) → SSE streaming → max 4000 token
Penilaian Kendiri (Self-Evaluation)
Selepas jawapan dihasilkan, model pantas menilai kualiti jawapan berdasarkan 3 dimensi: Relevan (adakah menjawab soalan?), Tepat (adakah berdasarkan dokumen?), dan Lengkap (adakah cukup terperinci?).
GPT-4o-mini → skor 1-5 × 3 dimensi
Cadangan Susulan (Follow-up Suggestions)
Model pantas menjana 3 soalan susulan yang berkaitan berdasarkan konteks perbualan. Membantu pengguna meneroka topik lebih mendalam.
GPT-4o-mini → 3 soalan cadangan

KESELAMATAN & KUALITI

Kawalan Kadar
Rate Limiting
Had 30 permintaan seminit bagi setiap IP. Dilaksanakan pada lapisan aplikasi dan Nginx.
Pengesahan URL
URL Verification
Hanya URL dari sumber web yang disahkan dibenarkan dalam jawapan. LLM tidak boleh mereka URL.
Penilaian Kendiri
3-Dimensi Scoring
Setiap jawapan dinilai secara automatik: Relevan (1-5), Tepat (1-5), Lengkap (1-5). Skor rendah memberi amaran kualiti.
Anti-Halusinasi
Source Grounding
Arahan sistem mewajibkan rujukan seksyen dokumen. Jika maklumat tidak dijumpai, chatbot akan menyatakan terus terang.
